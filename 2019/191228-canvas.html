—— 12.29 更新

　　今天早起清完暖暖日常就开始逐条排查冲突的 JS，结果一上来就发现 page 里的 canvas 即使不加 layout:false 也能把图画出来，排查了一番发现……产生冲突的是我自己写的 emoji 插件。
　　Next 对不起！！！！！ :-cmx-02-:
　　主要问题大概就是我之前偷懒给 emoji 设置的生效区域是 id="posts" 的整个大元素，里面包含了标题和正文还有日期分类 tag 之类的一大堆子元素，当然也包括我加进去的那个 canvas，然后 canvas 里的 drawIMG() 被延时到 emoji 替换完毕之后才开始执行？然后这个时候的 canvas 已经是 innerHTML 之后的新元素了，之前用 JS 获取的画布就失效了……大概是这个原理……
　　解决办法就是把 emoji 的替换范围改成了 posts 以下的所有 p 类元素，保证了 canvas 画布不被替换，就是以后在居中文本里写 emoji 可能会有点费劲了 emmmm
